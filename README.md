# CORA: Caption-Offset Retrieval for Audio

This repository is the public experiment-code package for the CORA paper. It
contains the setup, embedding, and diagnostic scripts needed to reproduce the
reported experiment results from the CORA test split.

The repository does not ship generated experiment outputs, figures, model
checkpoints, or third-party model repositories. The setup script downloads only
the `test` split from the Hugging Face dataset [`msnowchanj/CORA`](https://huggingface.co/datasets/msnowchanj/CORA); model weights
must still be prepared manually because they come from separate model releases.

## Repository Contents

| Path | Purpose |
| --- | --- |
| `scripts/setup_cora_test_split.py` | Downloads `msnowchanj/CORA` test split and writes the local JSONL/audio layout. |
| `scripts/embed_eq_renew_caches.py` | Builds embedding caches from the downloaded test split and local model checkpoints. |
| `scripts/run_paper_experiments.py` | Runs the paper diagnostics and prints the result reports. |
| `scripts/run_*.py` | Individual diagnostic scripts for specific paper tables/reports. |
| `src/clap_eval/` | Lightweight wrappers around the evaluated CLAP-family models. |
| `src/cora_repro/` | Verification and result-summary utilities. |
| `docs/` | Data, model, experiment, and reproducibility notes. |

Generated artifacts are written under `data/`, `results/`, and `experiments/`.
Figures are not generated by this release package.

## Installation

```bash
uv sync
```

Without `uv`:

```bash
pip install -e .
```

The lockfile is included for reproducibility, but PyTorch and CUDA wheels may
still need platform-specific adjustment.

## 1. Download CORA Test Split

Download only the Hugging Face `test` split and convert it to the local layout
used by the experiment scripts:

```bash
uv run python scripts/setup_cora_test_split.py
```

Default output:

```text
data/cora/test/eq_by_clip.jsonl
data/cora/test/audio/
```

Use `--overwrite` to replace an existing local copy.

## 2. Prepare Model Checkpoints

All model code and checkpoints are local-input only. Download or clone them
manually, then keep the default paths in `config.yaml` or update the paths for
your machine.

| Model | Default local inputs |
| --- | --- |
| LAION-CLAP | `models/laion-clap-htsat-unfused/` |
| M2D-CLAP | `models_third_party/m2d/` and `models/m2d/checkpoint-30.pth` |
| MGA-CLAP | `models_third_party/MGA-CLAP/` and `models/mga/mga-clap.pt` |
| MS-CLAP | installed `msclap` package and `models/msclap/msclap_2023.pt` |
| OEA / RobustCLAP | optional external retriever inputs described in `docs/MODELS.md` |

LAION-CLAP is loaded with `local_files_only: true` by default. The experiment
scripts should not require `HF_TOKEN` or `HUGGINGFACE_HUB_TOKEN` for model
loading.

Check the downloaded data and configured model paths:

```bash
uv run python scripts/check_local_inputs.py
```

## 3. Build Embedding Caches

Build the base CLAP caches:

```bash
uv run python scripts/embed_eq_renew_caches.py \
  --eq-jsonl data/cora/test/eq_by_clip.jsonl \
  --audio-dir data/cora/test/audio \
  --only laion msclap m2d mga
```

Optional external retriever caches can be built after preparing the OEA or
RobustCLAP repositories and checkpoints locally:

```bash
uv run python scripts/embed_eq_renew_caches.py \
  --eq-jsonl data/cora/test/eq_by_clip.jsonl \
  --audio-dir data/cora/test/audio \
  --only oea_qwen3b_cl oea_qwen3b_ac robustclap
```

Cache defaults:

```text
results/eq_renew/vanilla_multiclap_eq_embedding_cache/
results/eq_renew/single_model_invariance_embedding_cache/
results/eq_renew/external_retriever_embedding_cache/
```

## 4. Run Result Diagnostics

Run the main paper diagnostics from prepared caches:

```bash
uv run python scripts/run_paper_experiments.py
```

The wrapper creates result tables and `REPORT.md` files under `experiments/` and
prints the generated `REPORT.md` contents. It does not generate figures.

Individual result scripts:

| Result diagnostic | Script | Default output |
| --- | --- | --- |
| Source-caption anchor diagnostic | `scripts/run_source_caption_anchor_diagnostic.py` | `experiments/source_caption_anchor_diagnostic/` |
| Source-caption projection analysis | `scripts/run_source_caption_projection_analysis.py` | `experiments/source_caption_projection_analysis/` |
| TBMD-guided view intervention | `scripts/run_tbmd_guided_view_intervention.py` | `experiments/tbmd_guided_view_intervention/` |
| External retriever core diagnostic | `scripts/run_external_core_diagnostic.py` | `experiments/external_retriever_core_diagnostic/` |
| RankDrop token saliency audit | `scripts/run_rankdrop_token_saliency.py` | `experiments/rankdrop_token_saliency/` |

Use `--include-external` or `--include-token-saliency` with
`scripts/run_paper_experiments.py` to include the optional slower diagnostics.

## Checks

Verify the public source layout:

```bash
uv run cora-verify
```

Summarize generated local result artifacts:

```bash
uv run cora-summarize
```
